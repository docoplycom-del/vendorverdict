Built for lightweight procurement decisions, not black-box approval.

VendorVerdict is designed to help small teams make SaaS buying decisions with a clearer evidence trail, ranked risks, and follow-up questions before client or business data is stored in a tool.

Evidence first

Reports show the source assumptions used for each vendor, including official-source targets, fallback evidence, and live-check confidence where enabled.

Human approval

VendorVerdict does not automatically approve tools. It produces a procurement artifact for a human buyer, founder, consultant, or operator to review.

3

Operational controls

The production instance uses authentication, SQLite persistence, PDF/Markdown exports, backups, monitoring, and alert hooks.

What VendorVerdict stores

Stored data

  • Saved report inputs and generated report text.
  • Public lead form submissions such as name, email, company, vendors, and use case.
  • Operational metadata such as notification status and report export references.

What to avoid

Do not submit sensitive secrets

  • No passwords, API keys, or private security documents.
  • No personal data beyond what is needed to request a pilot.
  • No confidential client data in a public demo request.

Pilot safety

Start with representative use cases.

The recommended pilot workflow is to test VendorVerdict with typical SaaS buying scenarios first, then decide what should become a repeatable procurement workflow.
